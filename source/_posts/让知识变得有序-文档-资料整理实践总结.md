随着时间的积累，无论是我们的备忘录、个人或团队知识库，还是电脑桌面/家居环境等各种地方都会变得无序。有一句名言：『我们淹没在信息中，却也处于信息饥渴中』。杂乱无章的环境和知识会降低我们的效率，使我们产生更多垃圾而无法沉淀有价值的东西。

作为一个团队，文档管理和分类更为重要，无序的知识管理会阻碍团队发展，而良好的文档管理可以提高团队效率、降低交接成本。

## 分类方法

### 常见方法

- **主题分类法**
  - **定义：** 根据需要进行选择，如按照团队划分、技术种类等维度，示例：『市场营销』，『产品开发』，『人力资源』。
  - **适用：** 团队/个人知识库、项目工作、技术工作。
  - **优点：** 语义直观，便于理解，支持深度细分
  - **缺点：** 主题交叉时易重复存储或导致管理混乱（如"AI医疗"应归属 `医疗` 还是 `AI` ？）
  - **注意**
    - **MECE原则：** 确保 **主题之间无重叠** ，且覆盖所有相关领域。
    - **避免过度细分****：** 分类控制在10个以内。

```bash
Atomkit/
├─ 代码管理
├─ 编译构建/
│  ├─ 环境搭建
│  ├─ 打包构建
│  ├─ 上线
├─ 技术方案
│  ├─ 小程序
│  ├─ 遥控器支持
│  ├─ 屏幕适配
```

- **日期分类法**
  - **定义：** 按文档创建/修改时间组织（年→ 季→月→周→日）。
  - **适用：** 日报/周报/会议纪要，同一主题下的时间线归档
  - **优点：** 分类结构简单，支持时间线追溯
  - **缺点：** 不支持主题查找，长期积累后同级文件过多

```bash
月报/
├─ 周报/
├─ 月报/
│  ├─ 2022年
│  ├─ 2023年
│  ├─ 2024年
│  ├─ 2025年
```

- **进度/任务状态分类法**
  - **定义：** 区分某一项目不同阶段的文件，适用项目管理

```bash
项目X/
├─ 进行中/
├─ 待审核/
├─ 已完成/
```

- **重要性分类**
- **文件类型分类**
- **地理位置分类**
- **PARA分类法**

### **PARA分类法**

**PARA** 是由生产力专家 **Tiago Forte** 提出的知识管理框架，旨在帮助个人和团队以 **行动为导向** 组织信息。其核心思想是：**知识管理的终极目标不是存储，而是推动行动**。

#### 分类方式

PARA 是四个英文单词的首字母缩写，代表信息分类的四个层级：

| **类别** | **英文全称** | **定义** | **示例** |
| ------- | ----------- | ------- | ------- |
| **P**rojects（项目） | 有明确目标和截止时间的任务 | 当前正在进行的短期任务 | - 撰写季度报告 - 开发新功能 - 策划营销活动 |
| **A**reas（领域） | 需要长期维护的责任范围 | 持续关注的领域或职责 | - 健康管理 - 财务管理 - 职业发展 |
| **R**esources（资源） | 可能用到的参考资料 | 有价值的素材或知识库 | - 行业报告 - 工具教程 - 书籍笔记 |
| **A**rchives（归档） | 已完成或不再活跃的内容 | 历史项目或过时资料 | - 去年项目文档 - 旧会议记录 |

#### 适用场景

1. **团队协作**

   - 适合：项目管理、跨部门知识共享。
   - 示例：团队共享的 `Resources/设计素材` 可被多个项目调用。

2. **文件系统整理**

   - 适合：电脑文件夹、云盘（如 Google Drive）。
   - 示例：将“客户合同”作为 Area，具体合同签署作为 Project。

#### **优点与局限**

- **优点**
  - **聚焦行动**：避免“知识囤积”，推动信息转化为结果。
  - **减少决策疲劳**：分类规则简单明确。
- **局限**
  - **依赖定期维护**：需定期清理和迁移文件（如项目完成后归档）。
  - **不适合超复杂知识库**：如学术研究可能需要更细分的分类法。

## 实践方案

1. 进行第一次文档整理

   1. 把所有资源放在 `未整理` 或 `临时文件夹` 目录
   2. 创建 `归档` 文件夹（归档文件夹下也可以有未整理目录）
   3. 敲定当前层级的归类方式，如：主题、时间、 PARA、状态等方法进行分类
   4. 从未整理目录取出一个文件

      - 没有意义，就删除
      - 已经失效，就放在归档文件的未整理目录
      - 其他就放到子主题目录下（子主题未整理，可以先放在子主题的临时文件夹）

   5. 可以自上而下、自下而上、或任选一个目录进行整理

2. 文档创建：统一创建到临时文件夹
3. 定期整理

   1. 有价值的东西花更多时间搞好，然后分类或归档
   2. 没价值的东西删除

## 心得

1. 实际操作时，**多种分类方式组合使用**

    ```bash
    主题分类
    ├─ 主题分类
    │  ├─ 时间分类
    │  ├─ Para分类
    ```

2. 团队的文档管理可以**先考虑主题**划分，然后针对每个主题采用合适的分类方法
   - 尽量**基于不变的事物进行分类**
      - **BAD：** 基于人的分类、可能会随团队调整而变化的内容
   - 主题下可以有子主题，参考如下

    ```bash
    主题A/
    ├─ 临时文件夹（未整理）/
    ├─ 子主题1
    ├─ 子主题2
    ├─ 归档
    ```

3. **临时文件夹**

    - 降低创建文件时的决策成本
      - 提高整理效率，先把文件都丢到临时文件夹，然后再逐级整理
      - 避免随着时间的推移，已整理过的内容变得混乱
      - 大部分创建的文件都没有写完，或者无价值

4. **归档文件夹**
   - 过期的知识和文档直接丢里边
   - 时间分类自带归档属性

5. 按**日/周/月例行的工作推荐使用时间分类**方法
   - 如 OKR、绩效、周报、月报等

6. 针对**具体项目/技术推荐采用PARA分类**法

## 示例

1. 个人知识库：主题+PARA+状态管理

    ```bash
    目录
    ├─ 我是谁
    ├─ 每日工作站/
    │  ├─ 临时文件夹/
    │  ├─ 工作TODO/
    ├─ 团队管理/
    │  ├─ 领域（Areas）
    │  ├─ 资源（Resources）
    │  ├─ 归档（Archives）
    ├─ 个人成长/
    ├─ 未整理/
    ```

2. 团队知识库：基于主题+时间

    ```bash
    团队管理/
    ├─ 前端技术图谱
    ├─ 团建/
    ├─ 规范/
    │  ├─ 上下班 & 休假 & 报备
    │  ├─ 上线规范
    │  ├─ 周会流程
    │  ├─ CaseStudy/
    │  │  ├─ 2023-12-25 场景详情页就执行添加组控保存场景因少传groupName导致场景灯控制异常
    ├─ 规划/
    │  ├─ 2025
    │  ├─ 2024
    │  ├─ 2023
    ├─ 休假/
    ├─ 人才/
    │  ├─ 人才评价标准
    │  ├─ 招聘
    │  ├─ 激励
    │  ├─ 成长计划
    ├─ 资产/
    ├─ 归档/
    ```
